如何在空間不足的 Mac 上安裝 Claude Code

今天一早想安裝 Claude Code，結果被一個錯誤訊息擋下來了——Mac 磁碟空間不足。這是我踏入工程世界的第一個障礙，記錄下來給有同樣困擾的朋友參考。

01發現問題：需要 10.57 GB

安裝程式提示至少需要 10.57 GB 的可用空間，但 Mac 只剩不到 5 GB。這比我預期的麻煩多了。

02用終端機找出佔空間的元兇

打開終端機（Terminal），掃描哪些資料夾最佔空間：

du -sh ~/* 2>/dev/null | sort -rh | head -20

發現 Library 最大，繼續往 Caches 裡挖：

du -sh ~/Library/Caches/* 2>/dev/null | sort -rh | head -10
03發現 Claude App 快取佔了 13 GB！

掃描結果讓我嚇一跳——之前裝的 Claude 桌面版快取竟佔了將近 13 GB。另外 Slack、WebEx 也各自佔了數 GB，三個加起來超過 16 GB。

04清除快取，成功釋出 10 GB

清除 Claude App 快取（先確認 Claude 桌面版已關閉）：

rm -rf ~/Library/Caches/Claude

清除 Slack 快取：

rm -rf ~/Library/Caches/com.tinyspeck.slackmacgap

清除 WebEx 快取：

rm -rf ~/Library/Caches/Cisco\ WebEx\ LLC

清完後總共釋出超過 10 GB，終於有足夠空間了！

05安裝 Homebrew（Mac 套件管理工具）

Claude Code 需要 Node.js 環境，先透過 Homebrew 來安裝。在終端機貼上：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

安裝完成後確認：

brew --version
06安裝 Node.js

透過 Homebrew 安裝 Node.js：

brew install node

確認安裝成功：

node --version
npm --version
07安裝 Claude Code，大功告成 🎉

環境準備好之後，用 npm 全域安裝 Claude Code：

npm install -g @anthropic-ai/claude-code

建立測試資料夾並啟動：

mkdir ~/Desktop/claude-test
cd ~/Desktop/claude-test
claude

看到 Claude Code 的提示符號出現，成功了！對工程小白的我來說，這一刻真的超有成就感 🌱

Jane 的觀點

「很多時候擋住我們的不是能力，是空間——磁碟的空間，也是心裡的空間。清掉不需要的，才能裝進新的。」

FAQ常見問題

Q：如何在 Mac 磁碟空間不足的情況下安裝 Claude Code？

A：先用終端機指令掃描佔空間的快取資料夾，清除 Claude App、Slack、WebEx 等應用程式的快取，通常可以釋出 10GB 以上的空間，再進行安裝。
